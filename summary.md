# 项目概要

这个文件夹是一个基于 Godot 引擎的 2D 游戏项目，项目名为 `2D_Games`。从当前目录内容看，它还处在早期原型或教程跟做阶段：已经配置了 Godot 项目、主场景、瓦片地图和一批美术/音频资源，但还没有看到角色、敌人、武器、碰撞、UI 或游戏规则相关脚本。

## 项目入口

- `project.godot` 是 Godot 项目配置文件。
- 项目使用 Godot 4.6 特性配置。
- 启动场景被设置为 `res://scene/node_2d.tscn`。
- 渲染上关闭了默认纹理过滤，适合像素风资源。
- 窗口拉伸模式为 `canvas_items`，说明项目主要面向 2D 画布内容。

## 主场景结构

主场景文件是 `scene/node_2d.tscn`，根节点名为 `Game`，类型是 `Node2D`。

场景里目前有三个核心节点：

- `Camera2D`：摄像机偏移为 `(128, 128)`，缩放为 `2x`，用于放大像素风画面。
- `GroundTileMapLayer`：地面瓦片层，使用普通瓦片和动态瓦片组成地图主体。
- `OverLayTileMapLayer`：覆盖层瓦片，使用动态瓦片资源，可能用于水面、装饰、遮挡或动态地形效果。

从 `.tscn` 内容看，地图已经通过 `TileSet` 和 `TileMapLayer` 搭建了一块 16x16 左右的测试区域，并且包含部分动画瓦片配置。当前主场景更像是一个地图搭建/瓦片系统测试场景，而不是完整可玩的关卡。

## 资源内容

`resources` 目录分为贴图、音频和字体。

### 贴图资源

`resources/texture` 下包含以下类型的像素风素材：

- `瓦片.png`：静态地形瓦片。
- `动态瓦片.png`：带动画帧的瓦片，已在主场景 TileSet 中使用。
- `源石虫.png`：看起来是敌人或怪物素材。
- `浮游炮.png`：可能是武器、炮台、跟随攻击单位或敌方单位。
- `爆炸特效.png`：爆炸动画或特效素材。
- `道具ui.png`：道具或状态栏 UI 素材。
- `W.png`、`icon.png`：可能是按键提示、图标或临时素材。

这些资源指向的是一个像素风俯视角动作/射击游戏方向。

### 音频资源

`resources/audio` 下包含多种西部牛仔风格音效和背景音乐：

- 背景音乐：`Journey of the Prairie King` 相关 mp3，包括 Overworld、The Outlaw、Final Boss & Ending。
- 音效：开枪、装弹、脚步、爆炸、死亡、怪物受击、怪物死亡、获得强化、隐藏/奖励音效等。

这些命名强烈暗示项目想复刻或参考《星露谷物语》里的小游戏 `Journey of the Prairie King`，也就是一个像素风、西部主题、俯视角射击生存/闯关小游戏。

### 字体资源

- `resources/font/IPix.ttf`：像素字体，适合用于菜单、得分、血量、道具说明等 UI 文本。

## 当前项目想做什么

综合项目配置、资源命名和主场景内容，这个文件夹的目标大概率是制作一个 Godot 版的像素风 2D 射击游戏原型。它可能参考《Journey of the Prairie King》的玩法：玩家在小地图中移动、射击敌人、拾取道具、触发爆炸或强化效果，并配合西部风格音乐和音效形成街机式关卡体验。

当前已经完成或准备好的部分：

- Godot 项目基础配置。
- 主场景入口。
- 摄像机和像素风显示设置。
- 地图瓦片层和动画瓦片配置。
- 敌人、武器、爆炸、道具 UI、字体、音效、背景音乐等素材导入。

当前还缺失的关键部分：

- 玩家角色节点和移动控制脚本。
- 射击、子弹、装弹、命中判定逻辑。
- 敌人生成、敌人 AI、血量和死亡逻辑。
- 道具拾取、强化效果、UI 显示。
- 音频播放逻辑。
- 关卡流程、胜负条件、暂停/重开等游戏状态管理。

## 目录里的异常内容

根目录下有一个名为 `%APPDATA%` 的文件夹，里面是 `@openai/codex` 相关文件。这不像 Godot 游戏项目自身内容，更像是某次工具或环境配置误把用户 AppData 路径写进了项目目录。分析项目意图时不应把它算作游戏代码或资源。

## 总结

这是一个 Godot 4 的像素风 2D 游戏项目雏形，核心方向是西部牛仔主题的俯视角射击小游戏。目前项目重点还停留在资源导入和地图场景搭建阶段，已经为后续实现玩家、敌人、射击、爆炸、道具和音效系统准备好了素材基础。
